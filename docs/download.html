Compiling From Source

  1. Check out the commit you wish to build in Git, or download and extract a ZIP/tarball of the latest development version from GitHub.
  2. cd into the downloaded FreeRCT directory and call: 
    
mkdir build
cd build
cmake ..        # See below for optional arguments.
make
make install    # Optional but recommended step.
  3. You need to have CMake and Make installed in order to do run these commands. CMake will check that all required libraries are installed. If this fails, you need to install the libraries it complains about, and retry until all dependencies are found.
  4. You can now start FreeRCT via freerct if you used make install; otherwise use ./build/bin/freerct or make run.

You can pass the following arguments to CMake:

Option Default Value Explanation
-DVERSION_STRING N/A Use the specified string as the build version. If not specified, the version is detected automatically from Git. If this also fails, the version defaults to 0.0.0-revdetect-broken.
-DASAN OFF Use AddressSanitizer (ASan). ASan is a library that detects invalid memory accesses at runtime and terminates the program with a full backtrace upon detecting one. This makes it a useful and recommended tool for developers and everyone who wants to assist with discovering bugs. End-users should leave this option disabled.
-DCMAKE_INSTALL_PREFIX /usr Directory where make install installs.
-DUSERDATA_PREFIX ~/.freerct (Windows) Directory where user data such as savegames is stored. Note that ~ character expansion in the USERDATA_PREFIX variable to the user home directory is performed by FreeRCT at runtime and should not be done by shell expansion to ensure that the executable can be used by multiple users.
~/.config/freerct (other)

Start FreeRCT with the --help switch to list the available command-line options.
